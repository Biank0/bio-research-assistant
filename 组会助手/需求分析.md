# 研究生组会汇报助手 — 需求分析

<sub><em>- For the girl in the lab</em></sub>

## 0. 背景

### 0.1 行业现状

学术组会（Group Meeting / Lab Meeting）是全球高校实验室通行的科研管理机制。在生命科学领域，组会通常以周或双周为周期，由课题组成员轮流汇报阶段性进展，接受导师和同行的反馈与质询。

组会的核心目的不是"展示结果"——导师和学生都清楚，实验结果不会每周都有突破。组会真正要验证的是研究者**持续的投入节奏和推进方向**：在做、在想、在迭代。

### 0.2 问题定义

当前学术组会汇报存在以下效率瓶颈：

1. **信息组织成本高** — 一周的实验记录、文献阅读、方案思考散落在笔记本、微信截图、电子文档中，汇报前需要大量时间做信息归拢和结构化
2. **表达深度与准备时间负相关** — 时间有限时，汇报容易退化为"流水账式罗列"，无法体现思考深度
3. **重复性劳动占比大** — PPT 框架搭建、文献检索整理、计划排期等环节高度模式化，但每周都在重复

### 0.3 产品定位

本产品定位为**学术组会汇报的效率工具**，目标是将汇报准备的核心环节（信息组织、结构化表达、文献辅助、提问设计）流程化和半自动化，降低"组织"的时间成本，让精力回到研究本身。

## 1. 用户画像

| 维度 | 描述 |
|------|------|
| 身份 | 生物方向研究生（直博/硕士/博士） |
| 研究方向 | Bridge RNA / ISCro4 可编程基因组重排技术 |

| 现实约束 | 时间有限（实验之余）、不会折腾服务器、需要简单触发即用 |

## 2. 场景分析

### 2.1 组会特征
- 频率：每周一次（周二）
- 时长：10-20 分钟汇报 + 讨论
- 听众：导师 + 同组同学
- 潜规则：不怕说"没做出来"，怕的是"看起来什么都没想"

### 2.2 典型困境
- 实验周期长，一周产出的硬数据（gel 图、测序结果等）可能只有 1-2 个
- 纯放数据 → 一页 PPT 讲完 → 导师觉得"没做事"
- 需要用**思考深度**和**方向规划**弥补数据量不足

### 2.3 导师真正想看到的
- 你在**主动思考**，不是被动等结果
- 你有**下一步计划**，知道为什么这么做
- 你在**读文献**，了解领域动态
- 你能提出**好问题**，说明你抓住了问题本质
- 你有**时间感**——"下周一做X，周三看结果"比"之后试试Y"强十倍

### 2.4 导师的期望方向
- 不满足于复现已有工作
- 期望发现新亚型（生信筛选）
- 期望用 AI 修改蛋白提高效率
- 期望创新性实验设计

## 3. 解决方案设计

### 3.1 技术方案选型

#### PPT 生成工具调研

| 工具 | 类型 | 优缺点 | 结论 |
|------|------|---------|------|
| **PptxGenJS** (npm) | Node.js 库 | ✅ 已集成在 Hermes `powerpoint` skill 中<br>✅ 3500+ GitHub stars<br>✅ 支持图表/表格/图片<br>✅ 无需外部服务 | **首选：直接用** |
| **python-pptx** | Python 库 | ✅ 生态好<br>⚠️ 需要额外安装<br>⚠️ 排版控制较弱 | 备选 |
| **ppt.ai / presentations.ai** | SaaS | ✅ 效果好看<br>❌ 需要网络<br>❌ 无法本地定制 | 不采用 |
| **Hermes `powerpoint` skill** | 已有 Skill | ✅ 完整的创建/编辑/QA流程<br>✅ PptxGenJS + markitdown<br>✅ 设计规范（配色/字号/布局）<br>✅ 学术配色推荐 | **直接复用** |

**决策：直接使用 Hermes 已有的 `powerpoint` skill，无需造轮子。**

#### 文献检索工具

| 工具 | 用途 |
|------|------|
| Hermes `arxiv` skill | bioRxiv/PubMed/Semantic Scholar 检索 |
| PubMed E-utilities | 生物文献精准检索 |
| web_search | 补充性检索 |

#### 定时任务（Cronjob）方案

| 任务 | 频率 | 内容 |
|------|------|------|
| 周日提醒 | 每周日 14:00 | 提醒用户"该准备组会了"，附上快捷入口 |
| 文献监控（可选） | 每周一次 | 自动检索 Bridge RNA 领域新发表文献，推送摘要 |
| 实验记录整理提醒（可选） | 每周五 | 提醒用户记录本周实验情况 |

**实现方式：** Hermes cron job + iMessage 或 Apple Notes 推送

### 3.2 首次使用

第一次使用时，完成一次简单配置：

```
三个问题：
1. 你希望生成的文件（大纲、PPT等）存在哪个文件夹？
2. 你的研究方向/课题是什么？（一句话就够）
3. 组会频率？（每周/每两周）
```

配置完成后记住，下次直接开始。

### 3.3 整体交互流程

```
用户说："准备组会"
    ↓
（首次 → 三个配置问题 → 记住）
    ↓
展示功能菜单：

    ┌─────────────────────────────┐
    │  准备组会 — 你想做什么？      │
    ├─────────────────────────────┤
    │  1. 📚 文献检索与总结        │
    │  2. 📊 PPT 准备（大纲+生成） │
    │  3. ❓ 策略性提问准备         │
    │  4. 📝 实验进展整理          │
    │  5. 🚀 全流程一键搞定        │
    └─────────────────────────────┘
    ↓
进入所选模块 → 针对性提问 → 输出产物
```

### 3.4 功能模块详解

#### 模块 1：📚 文献检索与总结

**进入后的对话：**
- 本次想检索什么方向？（可以和你正在做的实验不同）
- 时间范围？（近一周 / 近一个月 / 不限）
- 有没有想追踪的特定课题组或作者？

**产出：**
- 筛选 2-3 篇最相关的文献
- 每篇一页纸总结：核心发现 + 对你课题的潜在启发
- 可直接作为 PPT"文献调研"页的素材

#### 模块 2：📊 PPT 准备（大纲+生成）

**进入后的对话：**
- 这周做了什么实验？（哪怕只是"跑了个 PCR 没出来"）
- 有什么结果或数据？
- 遇到了什么问题？
- 下周打算做什么？

**产出：**
- 8-12 页结构化大纲
- PPT 文件（学术简洁风格，Ocean Gradient 配色）
- 每一页的演讲备注（怎么讲、讲多久）

#### 模块 3：❓ 策略性提问准备

**角度一：实验实际问题**
> "我观察到 X 现象，已经排除了 A 和 B 的可能，我的假设是 C，打算用 D 验证——老师您觉得这个方向对吗？"

**角度二：文献前沿探索**
> "最近看到一篇关于 X 的文章，他们用 Y 方法实现了 Z，我觉得可能适用于我们的某环节——老师您怎么看？"

#### 模块 4：📝 实验进展整理

**产出：**
- 结构化实验记录（日期 / 目的 / 方法 / 结果 / 结论）
- 失败实验的 troubleshoot 思路
- 可直接导入 PPT 的素材块

#### 模块 5：🚀 全流程一键搞定

按顺序：素材收集 → 文献检索 → 大纲生成 → PPT 输出 → 策略性提问

### 3.5 "数据少"的破解策略

| 这周的情况 | 怎么讲 | 展示的能力 |
|------------|--------|-----------|
| 实验失败了 | 详细分析失败原因、排错路径 | 排错能力 |
| 在等结果 | 展示实验设计逻辑、为什么这样设计 | 规划能力 |
| 重复实验 | 对比不同条件的优化过程 | 严谨性 |
| 这周没做实验 | 文献综述 + 新方案设计 | 学习能力 |

**核心逻辑：导师要的不是数据本身，是数据背后的思考。**

## 4. 非目标（不做什么）

- ❌ 不帮你编造数据或美化失败实验的结果
- ❌ 不替代你对实验的理解——素材必须你提供，工具负责结构化和扩展
- ❌ 不要求你有编程基础或搭建任何环境
- ❌ 不做过度学术化的表达——保留你自己的话

## 5. 体验目标

> 周日下午花 15 分钟对话，拿到完整大纲和 PPT 初稿。晚上只需要微调排版、加上自己的实验图，十点之前搞定。

### 最终感受
不是在"用工具"，而是在跟一个了解你课题的学弟聊天——你提供这周的实验事实，他负责帮你把零散信息组织成一个有逻辑、有深度的故事。

## 6. 成功标准

- 冷启动配置 ≤ 1 分钟，之后无感
- 即使一周只有 1 个实验结果，也能生成 ≥ 8 页有内容的 PPT
- 策略性提问能让导师说出"这个问题问得好"
- 全流程对话时间 ≤ 20 分钟
- 用户不需要懂任何技术概念

## 8. PPT 工具市场调研结论

经调研，市面上的 PPT 生成方案分三类：

1. **编程库（本地生成）**
   - PptxGenJS（Node.js）— 已集成在 Hermes powerpoint skill
   - python-pptx（Python）— 社区广泛使用，排版控制较弱
   - SkillsMP 上有 `python-pptx-generator` skill 可参考

2. **SaaS 服务**
   - ppt.ai、presentations.ai、Gamma.app — 在线生成，效果好看但不可定制
   - 不适合需要本地运行、隐私敏感的学术场景

3. **Hermes 生态内已有方案**
   - `powerpoint` skill：完整的创建/编辑/QA 流程
   - 包含设计规范（学术配色推荐、字号规范、布局准则）
   - 支持 QA 验证（转图片检查排版）
   - **结论：无需引入外部工具，直接复用。**
