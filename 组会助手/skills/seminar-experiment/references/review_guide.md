# 回顾操作详细指南

本文件在 action=review 时按需加载，提供完整的回顾生成约束。

---

## 回顾流程

### Step 1: 确认回顾参数（一次性问）
需要确认：
- 时间范围（本周 / 本月 / 自定义）
- 回顾类型（seminar / mentor / self）
- 如果是 seminar，是否需要与 seminar-literature 的文献报告合并

### Step 2: 数据聚合
- 读取时间范围内所有 `logs/` 记录
- 读取相关 `threads/` 时间轴
- 读取 `rules.md` 中本周期新增的规则
- 如有 `analysis/` 报告也纳入
- 统计：总实验数、成功/失败/待验证/进行中 分布

### Step 3: 按类型生成报告

参考 `references/review_template.md` 的结构，按类型调整侧重：

---

## 类型差异

### seminar（组会汇报）

**侧重:** 进展展示 + 问题讨论

**结构:**
1. 概览（本周期做了什么，一段话）
2. 关键进展/里程碑（按重要性排序）
3. 实验数据摘要（核心结果，不展开所有细节）
4. 当前问题与阻塞（附自己的思考和已尝试方案）
5. 下一步计划（具体、有时间线）
6. 策略性提问

**策略性提问规则:**
- MUST 包含两个角度：
  1. 实验/技术问题：关于可行性、对照设计、条件优化（具体可答）
  2. 方向/前沿问题：关于研究趋势、替代路线、新可能性（开放讨论）
- SHOULD 提问基于实际遇到的困惑，不要为了有问题而编问题
- SHOULD 提问暗含自己的初步判断（"我倾向于X，但不确定Y是否更好"）

**与 seminar-literature 合并:**
- 如果同期有文献检索产出，可在报告中增加"文献启发"板块
- 将文献中的方法/发现与自己的实验问题对接
- 格式："{文献发现} → 对我们的启示：{具体实验环节}"

**时长控制:**
- 5分钟：只保留关键进展 + 核心问题 + 1个提问
- 10分钟：完整结构
- 15分钟：增加数据细节展示 + 方法对比讨论

---

### mentor（导师讨论）

**侧重:** 决策点 + 方向判断

**结构:**
1. 进展摘要（简洁，导师不需要所有细节）
2. 决策点（需要指导的 2-3 个问题，每个附 2-3 个选项）
3. 失败分析（诚实展示，附假设和验证计划）
4. 资源需求（试剂、设备、时间）
5. 时间规划（接下来 2-4 周的计划，要现实）

**决策点格式:**
```
### 决策: {question}

背景: {why_this_needs_a_decision}

选项 A: {description}
  - 优势: ...
  - 风险: ...
  - 时间成本: ...

选项 B: {description}
  - 优势: ...
  - 风险: ...
  - 时间成本: ...

我的倾向: {which_and_why}
```

---

### self（自我复盘）

**侧重:** 经验提炼 + 效率反思

**结构:**
1. 做了什么（事实陈列）
2. 有效的实验 vs 浪费的实验（哪些本可以不做）
3. 新增经验规则
4. 时间分配回顾（哪些实验占了过多时间）
5. 大方向反思（是否需要调整课题侧重）
6. 下周期优先级

**诚实性规则:**
- MUST NOT 美化失败
- MUST 标注"沉没成本"实验（做了很多但可能方向错了的）
- SHOULD 区分"有信息量的失败"和"无效重复"

---

## 回顾约束（强制）

- MUST 基于实际记录生成，不凭记忆补充未记录的实验
- MUST 进展和问题同等重要——不要只报好消息
- MUST "下一步"具体可执行，不写"继续优化"这类空话
- MUST NOT 在记录不足时生成虚假的完整回顾——标注"本周期记录较少，以下基于已有 N 条"
- SHOULD 回顾结束后建议用户补记遗漏的实验

---

## 产出规范

- 输出默认中文，术语保留英文
- 展示报告全文 → 用户确认 → 写入 `{experiment_dir}/reviews/{date}_{type}.md`
- 如是 seminar 类型，额外提供 bullet points 版本（可直接贴 PPT）
