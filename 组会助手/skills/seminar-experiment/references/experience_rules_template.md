# 经验规则模板

```markdown
# 经验规则库

## 规则索引

| ID | 规则名 | 课题 | 置信度 | 来源实验数 |
|----|--------|------|--------|-----------|
| R001 | {title} | {project} | 高/中/低 | {n} |

---

## R001: {rule_title}

**来源**: 
- [{date_1}] {experiment_1} — {关键观察}
- [{date_2}] {experiment_2} — {验证/反例}

**规则表述**: 
当 {condition} 时，{outcome}。

**适用条件**:
- {scope_1}
- {scope_2}

**置信度**: {高/中/低}
- 高: 3次以上重复验证
- 中: 2次验证或强机理支撑
- 低: 单次观察，待验证

**反例/例外**:
- {如有，记录在什么条件下规则不成立}

**相关规则**: R{xxx}

**状态**: 活跃 / 待验证 / 已修正 / 已废弃
```

## 经验规则设计原则

1. **条件句格式**: "当X时，Y更可能" — 不写"永远要Z"
2. **来源可追溯**: 每条规则必须链接到具体实验记录
3. **置信度诚实**: 一次观察就是低置信度，不要因为"合理"就标高
4. **可证伪**: 规则必须可以被未来实验推翻——写清什么情况下规则失效
5. **渐进更新**: 新实验可以升级/降级/废弃已有规则
6. **不删除**: 废弃的规则不删除，改状态并注明原因（历史信息有价值）
