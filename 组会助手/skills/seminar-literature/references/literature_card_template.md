# 文献精读卡模板

```markdown
# 文献精读卡

## 输入覆盖范围
- 来源: {PDF / bioRxiv / PubMed abstract / 仅标题}
- 覆盖级别: 🟢 全文 / 🟡 摘要 / 🔴 仅标题
- 边界说明: {如有未覆盖部分，在此声明}

## 基本信息
- **标题**: {title}
- **作者**: {authors}
- **期刊/预印本**: {journal}, {year}
- **DOI/链接**: {doi_or_url}

## 研究问题
- 论文试图解决什么问题？
- 为什么这个问题重要？（在领域内的位置）

## 核心方法
- 整体思路:
- 关键技术/工具:
- 与已有方法的核心差异:

## 实验设置
- 模型系统/物种:
- 关键实验手段:
- 对照设计:
- 验证方法:

## 主要结果
- 论文声称的核心发现:
- 证据是否充分支持 claim:
- 关键数据/图表:

## 优势
- 方法层面:
- 实验设计层面:
- 研究价值层面:

## 局限
- 证据不足处:
- 适用范围限制:
- 可能的失败场景:

## 与当前课题的关联
- 相关度: ⭐⭐⭐⭐⭐ (1-5)
- 具体关联点: {不要泛泛写，要具体到哪个实验/哪个机制}
- 可直接借鉴: {具体模块或方法}
- 可对比验证: {什么假设可以用类似方式验证}

## 阅读优先级
- 标签: 值得精读 / 值得速读 / 可暂缓
- 判断理由:

## 发散思考
1. {基于本文方法或发现，提出一个"如果……会怎样"的延伸问题}
2. {基于本文局限或未回答的问题，提出一个可能的新实验方向}
```

## 使用说明

- 输入不足时，优先保留结构，不要为了填满而虚构。
- 如果只有摘要，"实验设置""主要结果"写得更保守，明确标注推测。
- "与当前课题的关联"是最重要的部分——这决定了这篇文献对组会汇报的价值。
